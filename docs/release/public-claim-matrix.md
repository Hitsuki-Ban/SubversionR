# Public Claim Matrix

This matrix defines what SubversionR may claim for the Windows `win32-x64` Beta package and later public release work. It prevents fixture evidence from being broadened into product claims.

The current Beta package claim is limited to Marketplace pre-release distribution and local file-backed Windows `win32-x64` SVN workflows after the installed VSIX and state-engine packaging gates pass. Stable-channel publication, artifact signing/signed source-to-binary provenance, previous-stable rollback, broad remote/auth matrices, cross-platform packages, coverage-guided fuzzing, merge, merge preview, and mergeinfo stay outside the Beta claim.

## Status Vocabulary

- claimed: product behavior may be described as supported after release gates pass.
- fixture-only: evidence exists only for a local test fixture or narrow gate.
- deferred: planned or required later, but not part of the current public claim.
- unsupported: not planned for the current release scope.

## Repository Transports

| Transport | Status | Current Evidence | Public Claim Boundary |
| --- | --- | --- | --- |
| Local working copy from file-backed repositories | claimed | Native bridge, daemon, extension, and installed local Source Control fixture evidence | Core local SVN workflows are the primary Windows Beta product path once the installed E2E and state-engine packaging gates pass. |
| `svn://` localhost authenticated fixture | fixture-only | Source-built `svnserve` fixture for credential broker operations | Fixture evidence only; public server compatibility requires broader auth/network testing. |
| localhost HTTPS DAV | fixture-only | Source-built Apache HTTP Server/mod_dav_svn fixture for HEAD content and update through certificate broker | Does not imply arbitrary HTTPS SVN servers are supported. |
| Arbitrary HTTPS SVN servers | deferred | No public release matrix evidence | Requires non-localhost TLS, auth, proxy, certificate, failure, and performance gates. |
| `svn+ssh` | deferred | No product gate | Requires tunnel command security design and Workspace Trust review. |
| File and directory externals over mixed transports | deferred | Partial discovery/status planning only | Requires externals isolation and auth-failure gates before public claim. |
